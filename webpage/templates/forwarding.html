{# Topics reference panel #}

Your broker subscribers should listen to these topics (gateway publishes only — no subscriptions):

TopicDescriptionDirection
metacrust/{source}/{device_id}/{metric} One message per metric per interval PUB
metacrust/{source}/{device_id}/status Device status changes (online / offline / error) PUB

Message format (JSON):

{
  "device_id":  "sensor_01",
  "source":     "rs485",
  "metric":     "temperature",
  "value":      24.5,
  "unit":       "°C",
  "quality":    "good",
  "ts":         1714567890123
}

{source} = rs232 / rs485 / modbus_tcp  ·  {device_id} = device identifier  ·  {metric} = metric name (e.g. temperature, humidity)
